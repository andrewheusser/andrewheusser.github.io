TopiKhan (topic modeling + Khan Academy)¶

Exploring the topic space of Khan Academy videos to make learning more efficient¶

The overarching goal of this project is improve the efficiency of learning from Khan Academy videos. Our primary approach is to derive a 'knowledge profile' for each video/course-- a quantitative summary of the content present in the course derived from linguistic models applied to the course transcripts. Then, we’ll derive a 'knowledge fingerprint' for the learner, i.e. an estimate of the student’s current knowledge state based on quiz performance. Finally, we integrate these models to create algorithms that adapt the trajectory of a course to minimize the time it takes for a student to reach a 'target' knowledge profile. Below is a brief overview of some highlights from the project so far.

Load in the topic models¶

Using the the Khan Academy and Youtube API, I downloaded course transcripts for as many videos as I could. For each course, I concatenated the transcripts, and fit a topic model to each course. The result is a courses (4128) by topics (100) matrix, where each row is a vector of topic weights for each Khan Academy course. The colors represent the weights for each topic (i.e. how much of a particular topic was present in the course).

Plot the topic models for each course using Hypertools¶

High dimensional data are difficult to visualize. To deal with this, we developed a toolbox for high-dimensional data visualization and manipulation. The toolbox uses dimensionality reduction (specifically, principal components analysis) to embed high-dimensional data in a lower dimensional space, and we're expanding it to support other dimensionality reduction approaches.

Below, I'm plotting the topic model for each course as a 3D point (i.e. the first 3 principal components for the course as a coordinate), colored by the field of study. This will help us get a sense as to whether the the topics models are structured with respect to the kind of information that is being presented in the video. For example, you might expect the topic models for math courses to be more similar to each other than to say, an Arts and humanities course.

In [14]:
legend = list(sorted(set(group), key=group.index))

Interestingly, there is definitely structure in the plot above: courses that are conceptually similar to each other have more similar topic models. Another (perhaps, coarser) way to visualize this structure is to look at the correlation matrix for the fields of study. Below, I'm computing the average topic model for all courses in a given field, and then correlating those averaged topic vector over each field of study. This correlation matrix represents the similarity of each field of study to other fields of study.

Closing thoughts¶

While the above analyses are interesting, this is really only the tip of the iceberg. I'm also beginning to explore other related questions, such as:

1) Does the "scope" of the topic model used change the similarity structure among courses/fields of study? For example, does a topic model trained on many Wikipedia articles vs. only Khan Academy videos make everything look more similar? Likewise, does a topic model trained on a subfield distort the global similarity structure, emphasizing similarity in some dimensions over others?

2) Can we use dynamic topic models to capture the "trajectory" of a given course through "topic space"? For instance, if we break each video into smaller chunks and estimate topic models for each chunk, can we plot the path that the course travels? Do courses revisit topics in some structured way, and does this relate to the "quality" of the course? Do different students prefer different trajectory structures, and could this be used to personalize the selection of educational materials for the student?

While the structure described above is interesting in of itself, my greater hope is that these techniques may be used to improve personalized learning algorithms. In the same way that one can create a topic model for a particular video/course, it is also possible to model student knowledge as a vector of weights representing a student's proficiency with respect to a particular topic. Combining these two techniques has the potential to drastically improve personalized learning.

For example, if you can estimate a student's "knowledge fingerprint" (i.e. vector of weights representing topic knowledge), then you can make inferences about their likely performance on any other course by comparing the student's fingerprint to the topic vector for that course. This could be used to create adaptive learning programs that maximize the potential for learning on any given course, or custom automated video curations that teach a combination of skills.

Using content modeling to personalize and optimize learning¶

Below is a rough outline of how this could all fit together. Topic modelling could be used to infer the content of courses and quiz questions by modelling the transcripts and quiz text, respectively. It's then possible to estimate a student's "knowledge fingerprint" by leveraging their performance on the quiz. The more questions answered, the better the model of the student's knowledge state. Finally, one can select future educational materials to maximize the potential for knowledge gain with respect to a 'target' knowledge state.
